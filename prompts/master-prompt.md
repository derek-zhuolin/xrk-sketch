# 小红书封面 · 手绘笔记本风格 SVG · Master Prompt

> 把整段发给 Claude，只替换最底部【任务】section 的 `{}` 占位符。

---

请生成 SVG，画布 **1242×1660（小红书封面 3:4 竖版高清）**。
风格：手绘笔记本素描 + 水彩点缀（Anthropic 官方 illustration 风格 / 建筑师速写本）。

## 字体引入（必做）

在 `<svg>` 的 `<defs>` 里加：
```html
<style>
  @import url('https://fonts.googleapis.com/css2?family=Ma+Shan+Zheng&family=Architects+Daughter&display=swap');
</style>
```

- 中文：`font-family="'Ma Shan Zheng', cursive"`
- 英文/数字：必须单独用 `<tspan font-family="'Architects Daughter', cursive">` 包
- 例：`<text>AI 围观营</text>` 写成
  `<text><tspan font-family="'Architects Daughter'">AI</tspan> 围观营</text>`

## SVG 技法（必做，缺一不可）

1. **背景**：`#F5F0E6` 奶白旧纸色 + feTurbulence 噪点 overlay (opacity 0.08)，覆盖整个 1242×1660
2. **手抖 filter**：定义 `<filter id="rough">` 用 `feTurbulence(baseFrequency=0.02, numOctaves=3)` + `feDisplacementMap(scale=2)`，所有线条/矩形/路径加 `filter="url(#rough)"`
3. **主线条**：`#2C2520`（暖黑），stroke-width 1.5-2.5 随机变化，stroke-linecap="round"
4. **物体阴影**：定义 `<pattern id="hatch1">`(45°) 和 `<pattern id="hatch2">`(-45°)，crosshatching 叠加，**禁止纯色填充**
5. **标签色块**：opacity 0.6 + rough filter，色板：
   - 淡蓝 `#B5C5D8`
   - 淡绿 `#C8D4B8`
   - 淡紫 `#CCBFD8`
6. **散落"信息颗粒"**：小圆点 + 菱形 + 三角形，**总共 16-24 个**，色板：
   `#E8C9A8` / `#E8DCA8` / `#C5D5B8` / `#B8C5D6` / `#D0C5DC` / `#E8B8B0`
   opacity 0.6-0.8，部分用细虚线连向主物体

## 物件画法（关键，与多页 sketchnote 一致，不可降级）

- **不要画卡通简笔画**，要画到工业设计草图精度
- 每个物件至少**两层 crosshatching 影线叠加**
- 物体下方有椭圆投影（带模糊）
- 网格、纹理、铆钉、木纹这种细节要画出来
- 物体保持轻微等轴侧视图（3D 透视感）

## 默认不输出（除非用户明确要求）

封面优先表达**内容本身**，不绑定个人 IP 或系列编号。**默认全部不放**，仅当用户在【任务】中明确给出字段时才加：

- 署名 / 作者名 / @xxx
- 期号 / 序号 / No.XX / 第 X 期 / Vol.XX
- 专栏名 / 分类标签 / "XX 笔记"

留白比信息更重要——顶部和底部装饰区如果没有用户指定的内容，就**只放散颗粒**，不要为了"填满"而编个标签。

## 字号规范（小红书封面专属，比 PPT 大很多）

- **主标题 96-120**（必须够大，feed 缩略图也要看得清）
- 副标题 36-44
- 标签色块文字 32
- 底部 caption 28-32
- 页码/装饰小字 24

## 布局网格规范（1242×1660 竖版，必须严格遵守）

### 安全边距
- 上下各 80px，左右各 60px

### 区域划分（y 坐标）

| 区域 | y 范围 | 内容 | 规格 |
|---|---|---|---|
| 顶部装饰 | 80-130 | 右上散颗粒（左上分类**可选**，默认空）| 24px |
| **主标题** | **160-440** | 大标题，水平居中 | 单行 120px / 双行 96px |
| 副标题 | 460-540 | 钩子文案 + 手绘下划线 | 36-44px |
| **主视觉** | **580-1180** | 核心物件 | bbox ≤ 900×600，水平居中 |
| 标签色块 | 1220-1320 | 1-3 个关键词水彩色块横排 | 单块 220×80 |
| 底部装饰 | 1360-1580 | 散颗粒 + 1 行短句 caption（可选）| 28-32px |

### 主标题字号决策树
- 8 字以内 → 单行 120px，y 居中在 280
- 9-14 字 → 双行 96px，行 1 y=240，行 2 y=360
- 超过 14 字 → **拒绝执行**，要求用户砍字

### 主视觉缩放规则
- 物件 bbox 计算后，宽 ≤ 900，高 ≤ 600
- 超出用 `transform="translate(...) scale(...)"` 缩到 fit
- 物件中心 x = 621（画布水平中心）
- 物件下方留 40px 给标签色块

### 固定装饰
- 左上角小字 (60, 110)：分类/专栏名（**可选，默认留空**——用户没明确给就不放）
- 主标题下淡蓝波浪手绘下划线
- 副标题下小短下划线
- 右下角 (1100, 1540) 附近散几个小方格点

## 调试输出要求

生成前在 SVG 注释里输出实际坐标，例如：
```xml
<!-- 标题: "AI 围观营" 8字单行 120px y=280 -->
<!-- 副标题: y=510 -->
<!-- 主视觉: 望远镜 bbox=(171,580)-(1071,1180) scale=0.95 -->
<!-- 标签色块 3 个: x=200/511/822 y=1240 -->
<!-- 颗粒 18 个，分布: 顶部 5 / 主视觉周围 10 / 底部 3 -->
```
任何元素越界重新调整，不要交付越界版本。

## 输出格式

**只输出一个完整可渲染的 `<svg viewBox="0 0 1242 1660" width="1242" height="1660" xmlns="http://www.w3.org/2000/svg">`**，不要 HTML 外壳（HTML 包装由模板处理）。
