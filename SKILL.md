# 小红书封面 · 手绘笔记本风格

复用 `sketchnote-style` 的全部视觉 DNA（字体、手抖 filter、crosshatching、水彩色块、信息颗粒），但：
- **画布从 1600×900 横版改成 1242×1660 竖版**（3:4，小红书推荐高清尺寸）
- **布局从三栏 PPT 改成单页封面**（大标题主导 + 中部主视觉 + 底部装饰）
- **输出从纯 SVG 改成 HTML 包装**，内置 PNG 导出按钮

## 触发条件

- 「做个小红书封面」「xhs 封面」「这篇内容配个封面」
- 「手绘风封面图」「Anthropic 风格的小红书封面」
- 已有文章/选题，需要配封面图

## 何时不用

- 多图笔记的内页（用 baoyu-xhs-images 或 ljg-card 更合适）
- 真实摄影风格封面
- 需要人物头像的封面

## 工作流（严格执行）

1. **确认三件事**（用户没给的就问）：
   - 封面**主标题**（限 8-14 字，缩略图下也要能读懂）
   - 副标题/钩子（可选，限 16-24 字）
   - **主视觉**——按下面的「主视觉两条路径」选其一
2. **读 `prompts/master-prompt.md`**，填占位符，生成 SVG。
3. **读 `templates/cover.html`**，把 SVG 注入 `<!-- SVG_HERE -->` 标记位置。
4. **保存到 `~/Desktop/`** 下当前项目文件夹（按 Derek 的全局规则，不用 `/tmp/`）。文件名：`xhs-cover-{主题简写}-{YYYYMMDD}.html`。
5. **告诉用户**：双击 HTML 打开，点右上角「下载 PNG」按钮即可得到小红书可直接上传的封面图。

## 多版本对比（design shotgun 模式）

当用户**没明确说"只要一版"** 或者**主动要求看多个候选** 时，主动产出 2-4 个**结构差异最大**的版本，并合成一个 gallery HTML 让用户在一页里横向对比、各自下载。

### 何时触发
- 用户说"给我看几种" / "做几个版本" / "对比一下" / "在一个页面展示 N 张"
- 主题模糊或抽象，不确定走「物件路径」还是「关键词路径」时——主动 shotgun
- 用户对单版反馈"还行但说不上来" → 提议做对比版

### 排版差异化策略（4 种已验证的不同方向）
1. **实物 icon 版**——按"主视觉路径 A"，配一个精准物件（前提：物件能成立）
2. **关键词大字版**——按"主视觉路径 B"，中央一个 2-4 字核心词
3. **标题打散版**——主标题三段错位排版（Saul Bass 风），标题本身=主视觉
4. **卡片堆叠版**——3 张错落便签卡片堆叠（设计师工作台风），中央卡片承载关键词

每次至少给 2 个**结构本质不同**的方向（不是细节微调），不要堆同质方案。

### 实现方法

**单版输出**：用 `templates/cover.html` 模板。

**多版对比**：
1. 先生成 N 个独立版本 HTML（每个用单版模板）
2. 用 `templates/gallery-builder.py` 拼成一个 gallery：
   ```bash
   # 编辑脚本里的 variants 列表指定要对比的版本
   python3 ~/.claude/skills/xrk-sketch/templates/gallery-builder.py
   ```
3. gallery.html 特性：响应式 grid 布局、每张图独立下载/复制按钮、字体只下载一次共享缓存

## 主视觉两条路径（核心策略）

**先尝试路径 A，找不到合适的就走路径 B。不要硬凑。**

### 路径 A：精准实物隐喻（手绘物件 icon）

适合：物件能精准、不冲突地传达主题精神。

例：
- "AI 围观营" → 望远镜（直接的"观察"动作）
- "复利系统" → 齿轮组 / 老式计数器（机械积累）
- "认知拆解" → 棱镜 + 光谱（拆开看清）
- "高级品牌打造" → 皮革旅行箱（LV 起家做的就是箱包）

**反隐喻黑名单（看到这些组合立刻换路径 B）：**

| 主题词 | 不要用 | 原因 |
|---|---|---|
| 高级品牌 / 奢侈品 / 大牌 | 印章、章子、公文戳 | 传递作坊/官僚/法务感，与"高级"反义 |
| 创意 / 灵感 | 灯泡 | 被用滥的 cliché，立刻拉低质感 |
| 多任务 / 协作 | 章鱼、瑞士军刀 | 同上，烂大街图标 |
| 思维 / 洞察 | 大脑切片、齿轮人头 | AI 生成图典型伪深度 |
| 增长 | 火箭、向上箭头 | 创业 PPT slop |

**自检问题**："LV / Apple / Hermès 会用这个物件做封面吗？" 答案是"不会"就换。

### 路径 B：纯版式 / 关键词驱动（无 icon）

**触发条件（满足任一即用）：**
- 想了 30 秒以上找不到强契合的实物隐喻
- 主题是抽象概念（思维方式 / 方法论 / 价值观 / 哲学）
- 用户主动说"不要 icon"

**布局要点：**
- 主视觉位置（y 580-1180）放一个**核心关键词大字**（中文 200-260px / 英文 280-300px）
- 关键词必须满足：**与主标题、副标题不重复任何字**
- 关键词应承载主题灵魂，是内容的浓缩，不是装饰
- 配 2-3 条手绘装饰横线 + 几个几何点缀 + 颗粒
- 大量留白，参考 Pentagram / New Yorker 海报

**关键词提炼示例：**
| 主题 | 主标题 | 中心关键词候选 |
|---|---|---|
| GPT 2.0 帮你做品牌 | （含"做品牌"） | 破局 / 重写 / 升维（不能用"做"或"品牌"）|
| 普通人如何攒第一桶金 | （含"攒钱"） | 复利 / 守拙 / 慢富 |
| 一人公司怎么活下来 | （含"一人"） | 杠杆 / 长期 / 韧性 |

如果用户没主动给关键词，**主动从主题里提炼 1-2 个候选**让用户选，不要默认硬上。

## 风格 DNA（必做，与 sketchnote-style 一致）

### 字体
```html
<defs>
  <style>
    @import url('https://fonts.googleapis.com/css2?family=Ma+Shan+Zheng&family=Architects+Daughter&display=swap');
  </style>
</defs>
```
- 中文：`'Ma Shan Zheng', cursive`
- 英文/数字：用 `<tspan font-family="'Architects Daughter', cursive">` 单独包

### 手抖 filter
```xml
<filter id="rough">
  <feTurbulence baseFrequency="0.02" numOctaves="3" />
  <feDisplacementMap in="SourceGraphic" scale="2" />
</filter>
```
所有 path/rect/line/circle 加 `filter="url(#rough)"`。

### Crosshatching（物体阴影必用，不许纯色填充）
```xml
<pattern id="hatch1" patternUnits="userSpaceOnUse" width="4" height="4" patternTransform="rotate(45)">
  <line x1="0" y1="0" x2="0" y2="4" stroke="#2C2520" stroke-width="0.5" opacity="0.4"/>
</pattern>
<pattern id="hatch2" patternUnits="userSpaceOnUse" width="4" height="4" patternTransform="rotate(-45)">
  <line x1="0" y1="0" x2="0" y2="4" stroke="#2C2520" stroke-width="0.5" opacity="0.4"/>
</pattern>
```

### 色板
- 纸色 `#F5F0E6` + feTurbulence 噪点 (opacity 0.08)
- 主线条 `#2C2520`（暖黑）
- 标签三色：`#B5C5D8` / `#C8D4B8` / `#CCBFD8`（opacity 0.6）
- 颗粒：`#E8C9A8` `#E8DCA8` `#C5D5B8` `#B8C5D6` `#D0C5DC` `#E8B8B0`

## 小红书封面专属布局规则（1242×1660）

竖版 3:4，安全边距上下 80px、左右 60px。

### 默认不输出（保持通用性，硬规则）

封面优先表达**内容本身**，不绑定个人 IP 或系列编号。以下元素**默认全部不放**，仅当用户在本次对话中**明确给出对应字段**时才加：

- 署名 / 作者名 / @xxx
- 期号 / 序号 / No.XX / 第 X 期 / Vol.XX
- 专栏名 / 分类标签 / "XX 笔记"

留白比信息更重要——顶部和底部装饰区如果没有用户指定的内容，就**只放散颗粒**，不要为了"填满"而编个标签。

### 区域划分

| 区域 | y 范围 | 用途 | 字号/规格 |
|---|---|---|---|
| 顶部装饰 | 80-130 | 右上散颗粒（左上分类**可选**，默认留空）| 24px |
| **主标题区** | **160-440** | 1-2 行大标题，水平居中 | **96-120px**（必须够大，缩略图也要清楚） |
| 副标题/钩子 | 460-540 | 1 行小字 + 手绘下划线 | 36-44px |
| **主视觉区** | **580-1180** | 核心物件，水平居中，crosshatching 必带 | bbox ≤ 900×600，scale 适配 |
| 标签色块 | 1220-1320 | 1-3 个关键词水彩色块横排 | 块 220×80 |
| 底部装饰区 | 1360-1580 | 散颗粒 + 1 行短句 caption（可选）| 28-32px |

### 标题字号决策
- 8 字以内 → 单行，120px
- 9-14 字 → 2 行，96px
- 超过 14 字 → 让用户砍，不要塞

### 主视觉精度（与 sketchnote-style 一致，不可降级）
- **禁止简笔画**，画到工业设计草图精度
- 至少两层 crosshatching 影线叠加
- 物件下方淡椭圆投影 + 模糊
- 网格、纹理、铆钉、木纹细节必须有
- 轻微等轴侧视图（3D 透视感）

### 颗粒装饰
- 顶部右上角散 4-6 个
- 主视觉周围散 8-12 个，部分用细虚线连向主物体
- 底部散 4-6 个
- 总数控制 16-24 个

## HTML 包装与 PNG 导出（核心差异点）

读 `templates/cover.html`，把生成的 SVG 整段替换 `<!-- SVG_HERE -->` 标记。

模板已内置：
- 居中预览容器（白底 + 投影，模拟真实小红书 feed 卡片感）
- 右上角悬浮「下载 PNG」按钮
- 纯前端 SVG → Canvas → PNG 导出（用 `XMLSerializer` + `Image` + `canvas.toBlob`）
- 输出 PNG 尺寸 1242×1660，2x DPR 下载（保证清晰度）

## 调试输出要求

生成 SVG 前，在注释里输出每个区域的实际 y 坐标和 bounding box，例如：
```
<!-- 标题: y=160-380 (单行 120px), 副标题: y=460-510, 主视觉: 望远镜 bbox=(171,580)-(1071,1180) scale=0.95 -->
```
任何元素超出 box 重新调整，不交付越界版本。

## 失败排查

| 现象 | 原因 | 修复 |
|---|---|---|
| 标题在小红书 feed 缩略图里看不清 | 字号 < 96 | 加大到 96-120 |
| 物件像简笔画 | crosshatching 没叠两层 | 强制 hatch1 + hatch2 |
| 线条太规整 | 缺 `filter="url(#rough)"` | 全量加上 |
| 英文字体丑 | 没用 tspan 包 | 用 Architects Daughter tspan |
| PNG 导出空白 | 字体未加载完成就导出 | 模板已用 `document.fonts.ready` 等待 |
| 颜色过艳 | opacity 没控制 | 标签 0.6，颗粒 0.6-0.8 |

## 与 sketchnote-style 的关系

完全独立的两个 skill。共享视觉 DNA，但用途、画布、布局、输出格式都不同：
| | sketchnote-style | xrk-sketch |
|---|---|---|
| 场景 | 多页 PPT / 流程图 | 单张小红书封面 |
| 画布 | 1600×900 横版 | 1242×1660 竖版 |
| 布局 | 三栏 PPT | 单页封面（标题+主视觉+装饰）|
| 输出 | 多个 SVG | HTML（含 SVG + PNG 导出按钮）|
| 标题字号 | 56 | 96-120 |
